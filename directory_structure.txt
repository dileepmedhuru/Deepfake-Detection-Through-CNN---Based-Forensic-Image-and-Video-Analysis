QUICK START
───────────
1.  python setup.py                        # create dirs + initialise DB
2.  cd backend
3.  pip install -r requirements.txt        # installs all packages (including new ones)
4.  python app.py                          # starts on http://localhost:5000
    → Admin password printed ONCE on first run – change it via profile.html

EMAIL SETUP (OPTIONAL)
───────────────────────
1.  Enable 2-Step Verification on your Gmail account
2.  Generate an App Password: https://myaccount.google.com/apppasswords
3.  Update .env file:
    MAIL_USERNAME=your-email@gmail.com
    MAIL_PASSWORD=your-16-char-app-password    # remove all spaces
4.  Restart: python app.py

Without email setup, the app works fine — emails are just silently skipped.

ML TRAINING (OPTIONAL - for better accuracy)
──────────────────────────────────────────────
1.  python data_preprocessing.py --celebdf-dir ../Celeb-DF-v1 --output-dir ../processed_dataset
2.  python train_model.py --epochs 50 --lr 0.001
    → Uses EfficientNetB0 transfer learning (much better than old custom CNN)
    → Handles class imbalance with compute_class_weight
    → Expected accuracy: 88-94% (vs 75-82% with old model)
3.  python evaluate_model.py

ENVIRONMENT VARIABLES (.env)
─────────────────────────────
SECRET_KEY=<generate with: python -c "import secrets; print(secrets.token_hex(32))">
JWT_SECRET_KEY=<generate with: python -c "import secrets; print(secrets.token_hex(32))">
ADMIN_EMAIL=admin@deepfake.com                     # optional
ADMIN_PASSWORD=<strong password>                   # optional (auto-generated if not set)

# Email (optional)
MAIL_SERVER=smtp.gmail.com
MAIL_PORT=587
MAIL_USE_TLS=true
MAIL_USERNAME=your-email@gmail.com                 # leave blank to disable email
MAIL_PASSWORD=your-app-password                    # 16-char app password (no spaces)

# File cleanup
UPLOAD_RETENTION_DAYS=7                             # deletes files older than this (default: 7)

KNOWN LIMITATIONS
──────────────────
• Video analysis is synchronous (blocks server thread) — use Celery for production
• No real-time progress for video analysis — users see frozen progress bar
• SQLite has concurrency limits — use PostgreSQL for production
• No unit tests — add pytest for production
• Docker not included — manual deployment required

TROUBLESHOOTING
────────────────
Q: "ModuleNotFoundError: No module named 'flask_limiter'"
A: Run: pip install -r requirements.txt

Q: "Email not sending"
A: Check .env MAIL_USERNAME and MAIL_PASSWORD are filled correctly (no spaces in password)
   Check spam folder
   Enable 2-Step Verification and generate App Password from Google

Q: "Upload button not working"
A: Replace frontend/js/upload.js with the fixed version
   Hard refresh browser (Ctrl+F5 or Cmd+Shift+R)

Q: "Model in demo mode but I have cnn_model.h5"
A: Check backend console — it shows why model failed to load
   Ensure MODEL_PATH in .env points to correct location
